# テストデータ投入スクリプト

税務申告アプリケーションの動作検証用テストデータを投入するスクリプト集です。

## 📦 概要

以下のテストデータを MongoDB に投入します：

| データ種別 | 件数 | 説明 |
|-----------|------|------|
| 不動産物件 | 5件 | アクティブ3件、売却済み2件 |
| 給与所得 | 3年度 | 2023～2025年度の給与所得計算結果 |
| RSU所得 | 3年度 | 複数企業からの権利確定記録 |
| 譲渡所得 | 2件 | 売却済み物件の譲渡所得計算 |
| 不動産所得 | 9件 | 3物件 × 3年度の所得計算 |

---

## 🚀 使用方法

### 1. 全データを一括投入（推奨）

```bash
cd backend
npm run seed:all
```

### 2. 個別にデータ投入

```bash
# 不動産物件マスタデータ
npm run seed:properties

# 給与所得データ
npm run seed:salary

# RSU所得データ
npm run seed:rsu

# 譲渡所得データ
npm run seed:capital-gain

# 不動産所得データ
npm run seed:real-estate
```

## 🎯 動作検証シナリオ

### シナリオ1: 不動産管理の確認

1. アプリ起動後、「不動産管理」タブを開く
2. 5件の物件が表示されることを確認
3. 売却済み物件（D、E）が正しく区別されていることを確認

### シナリオ2: 給与所得の確認

1. 「給与所得」タブを開く
2. 3年度分の計算結果が表示されることを確認
3. 控除額が正確に計算されていることを確認

### シナリオ3: RSU所得の確認

1. 「RSU所得一覧」タブを開く
2. 3年度分のデータが表示されることを確認
3. 複数企業からの権利確定が正しく集計されていることを確認

### シナリオ4: 不動産所得の確認

1. 「不動産所得一覧」タブを開く
2. 年度を選択（2023～2025）
3. 3物件の所得データが表示されることを確認
4. CSV出力ボタンをクリックして、正常にダウンロードできることを確認

### シナリオ5: 譲渡所得の確認

1. 「譲渡所得」タブを開く
2. 「売却済み物件を取得」ボタンをクリック
3. 2件の売却済み物件が候補として表示されることを確認
4. 物件を選択して計算実行
5. 税額が正しく計算されることを確認

### シナリオ6: CSV/PDF出力テスト

1. 各モジュールのCSV出力機能をテスト
2. 日本語が正しく表示されることを確認（UTF-8 BOM）
3. Excel で開いて文字化けがないことを確認

## 🔧 トラブルシューティング

### MongoDB接続エラー

```bash
# MongoDB が起動しているか確認
ps aux | grep mongod

# .env ファイルの MONGODB_URI を確認
cat ../.env | grep MONGODB_URI
```

### データが重複する場合

各スクリプトは実行時に既存データを自動的に削除してから投入します。
重複の心配はありません。

### スクリプト実行エラー

```bash
# ts-node がインストールされているか確認
npm list ts-node

# 必要に応じて再インストール
npm install ts-node --save-dev
```

## ⚠️ 注意事項

- **本番環境では使用しないでください**
- テストデータは架空のものです
- 既存のデータは削除されます（各スクリプト実行時）
- MongoDB が起動している必要があります
